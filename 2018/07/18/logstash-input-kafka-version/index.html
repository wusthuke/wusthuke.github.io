logstash-6.2.3无法消费kafka-0.9.0

近期在对公司的ELK进行版本升级，kafka使用的版本为：kafka_2.11-0.9.0.1，logstash版本为：logstash-6.2.3，
在使用 logstash 对 kafka进行消费时，发现数据无法消费，kafka-server端报错。
将 kafka 升级到 kafka_2.11-0.10.1.1 后，数据正常消费。

由于各种原因，kafka不能及时升级，所以想办法从logstash端考虑解决该问题。通过 logstash官网
了解到，6.2.3使用的kafka版本为 1.0.0。

logstash-input-kafka 与 kafka的匹配关系

数据来源于： logstash-input-kafka changeLog

version kafka_version
logstash-inut-kafka-8.0.4 1.0.0
logstash-inut-kafka-6.3.3 0.11.0.0
logstash-inut-kafka-6.3.0 0.10.2.1
logstash-inut-kafka-6.2.2 0.10.1.1
logstash-inut-kafka-6.0.0 0.10.1.0
logstash-inut-kafka-5.0.4 0.10.0.1
logstash-inut-kafka-5.0.0 Support for Kafka 0.10 which is not backward compatible with 0.9 broker.
logstash-inut-kafka-4.0.0 0.9

通过上表对应关系，下载 logstash-input-kafka-4.2.0.gem

2.移除现有高版本 logstash-input-kafka
